# Stacks & Queues

Pilhas e filas são estruturas "abstratas": não importa como são implementadas por dentro, mas sim a **disciplina de acesso** que impõem. Essa disciplina — quem entra primeiro, quem sai primeiro — é o que as torna a ferramenta certa para problemas específicos.

## Stack (pilha): LIFO

Uma pilha segue **LIFO** — *Last In, First Out*. O último a entrar é o primeiro a sair, como uma pilha de pratos. Só se mexe no topo.

```js
const pilha = []
pilha.push(1)      // [1]
pilha.push(2)      // [1, 2]
pilha.push(3)      // [1, 2, 3]
pilha.pop()        // 3  → [1, 2]   (remove do topo)
pilha[pilha.length - 1]  // 2  (peek: olha o topo sem remover)
```

Em JavaScript, um array já é uma pilha eficiente: `push` e `pop` são `O(1)`. Todas as operações de pilha — push, pop, peek — são `O(1)`.

**Onde pilhas aparecem:**
- A própria **call stack** da recursão (dia 4).
- Verificar **parênteses balanceados**.
- Desfazer/refazer (undo/redo), avaliação de expressões, navegação "voltar".

```js
// Parênteses balanceados
function balanceado(s) {
  const pilha = []
  const par = { ')': '(', ']': '[', '}': '{' }
  for (const c of s) {
    if (c === '(' || c === '[' || c === '{') pilha.push(c)
    else if (c in par) {
      if (pilha.pop() !== par[c]) return false   // não casa com o topo
    }
  }
  return pilha.length === 0                       // sobrou aberto?
}
balanceado('([]{})')  // true
balanceado('([)]')    // false
```

## Queue (fila): FIFO

Uma fila segue **FIFO** — *First In, First Out*. O primeiro a entrar é o primeiro a sair, como uma fila de banco. Insere-se atrás (`enqueue`), remove-se na frente (`dequeue`).

```js
const fila = []
fila.push(1)       // enqueue → [1]
fila.push(2)       // [1, 2]
fila.shift()       // dequeue → 1  → [2]
```

**Cuidado de performance:** usar `array.shift()` para dequeue é `O(n)` — ele reindexa todo o array! Para filas com muitas operações, isso vira gargalo. Soluções:

```js
// Opção A: fila com ponteiro de cabeça (não reindexa)
class Fila {
  constructor() { this.itens = []; this.cabeca = 0 }
  enqueue(x) { this.itens.push(x) }           // O(1)
  dequeue() { return this.itens[this.cabeca++] } // O(1) amortizado
  get tamanho() { return this.itens.length - this.cabeca }
}
```

Opção B é usar uma **linked list** (dia 6) com referência a head e tail — enqueue e dequeue ficam `O(1)` de verdade. É um ótimo exemplo de como a estrutura interna influencia a performance.

**Onde filas aparecem:** o BFS (busca em largura) em grafos, escalonamento de tarefas, buffers de mensagens — qualquer "primeiro a chegar, primeiro a ser atendido".

## Deque: o melhor dos dois mundos

Um **deque** (*double-ended queue*) permite inserir e remover nas **duas pontas** em `O(1)`. Generaliza pilha e fila. É a base do padrão de janela deslizante com máximo/mínimo.

```text
addFront / removeFront        addBack / removeBack
        ←  [ a, b, c, d ]  →
```

## Padrões clássicos

**Fila usando dois stacks** — prova que você entende a diferença entre LIFO e FIFO. Empilhar em um e despejar no outro inverte a ordem duas vezes, recuperando FIFO:

```js
class FilaComStacks {
  constructor() { this.entrada = []; this.saida = [] }
  enqueue(x) { this.entrada.push(x) }
  dequeue() {
    if (this.saida.length === 0) {
      while (this.entrada.length) this.saida.push(this.entrada.pop())
    }
    return this.saida.pop()
  }
}
// dequeue é O(1) amortizado
```

> **Monotonic stack** — uma pilha que mantém seus elementos sempre crescentes (ou decrescentes), descartando o que "não serve mais". Resolve em `O(n)` problemas como "próximo elemento maior" que pareceriam exigir `O(n²)`:

```js
// Para cada elemento, o índice do próximo maior à direita
function proximoMaior(nums) {
  const res = new Array(nums.length).fill(-1)
  const pilha = []   // guarda índices, valores decrescentes
  for (let i = 0; i < nums.length; i++) {
    while (pilha.length && nums[i] > nums[pilha[pilha.length - 1]]) {
      res[pilha.pop()] = i
    }
    pilha.push(i)
  }
  return res
}
// O(n): cada índice entra e sai da pilha no máximo uma vez
```

> **Resumo:** Pilha é **LIFO** (mexe só no topo, tudo `O(1)`) — pense em recursão, balanceamento e undo. Fila é **FIFO** (entra atrás, sai na frente) — pense em BFS e escalonamento, mas evite `array.shift()` (`O(n)`): use ponteiro de cabeça ou linked list. **Deque** opera nas duas pontas em `O(1)`. Os padrões de ouro são **fila com dois stacks** e a **monotonic stack**, que resolve "próximo maior/menor" em `O(n)`.
