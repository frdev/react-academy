# Listas e Renderização Condicional

Quase toda tela é, no fundo, **uma lista de coisas**: produtos, mensagens, tarefas. E quase toda tela esconde ou mostra algo conforme uma condição: "carregando...", "nenhum resultado", um botão só para quem está logado. Em React, as duas coisas saem direto do JavaScript que você já sabe — `.map` (do Dia 4 do JS Essencial) para listas, e operadores lógicos para condicionais — combinados com JSX. Nada de sintaxe nova de template: é JavaScript dentro das chaves.

## Renderizar listas com map

Para transformar um array de dados numa lista de elementos, use **`.map`** retornando JSX. Como é React com TypeScript, tipamos os itens:

```tsx
type Fruta = { id: number; nome: string }

const frutas: Fruta[] = [
  { id: 1, nome: "Maçã" },
  { id: 2, nome: "Uva" },
  { id: 3, nome: "Pera" },
]

function Lista() {
  return (
    <ul>
      {frutas.map((fruta) => (
        <li key={fruta.id}>{fruta.nome}</li>
      ))}
    </ul>
  )
}
```

As chaves `{ }` abrem JavaScript dentro do JSX; `frutas.map(...)` devolve um array de `<li>`, e o React renderiza cada um.

## A prop key

Repare no **`key={fruta.id}`**. O React pede uma `key` única em cada item de uma lista para acompanhar a identidade de cada elemento entre renders — assim ele sabe o que mudou, foi adicionado ou removido, e atualiza só o necessário. Use um **identificador estável e único** (o `id` do dado). Evite o índice do array como key quando a lista pode mudar de ordem ou tamanho, pois isso confunde o React e pode causar bugs visuais. Sem `key`, o React emite um aviso no console.

## Renderização condicional

Para mostrar algo só sob condição, use os operadores que você já conhece, dentro das chaves:

```tsx
function Painel({ logado, qtd }: { logado: boolean; qtd: number }) {
  return (
    <div>
      {/* && : renderiza só se a condição for verdadeira */}
      {logado && <p>Bem-vindo de volta!</p>}

      {/* ternário : escolhe entre dois */}
      {qtd > 0 ? <p>{qtd} itens no carrinho</p> : <p>Carrinho vazio</p>}
    </div>
  )
}
```

- **`&&`** mostra o elemento quando a condição é verdadeira (e nada quando falsa);
- **ternário** (`cond ? a : b`) escolhe entre duas saídas.

Cuidado com o `&&`: se a esquerda for o número `0`, ele renderiza `0` na tela (lembra do falsy?). Para contagens, prefira `qtd > 0 && ...`.

Quando o componente inteiro depende de um estado (ex.: ainda carregando), o **early return** deixa o código mais limpo:

```tsx
function Perfil({ dados }: { dados: { nome: string } | null }) {
  if (!dados) {
    return <p>Carregando...</p>
  }
  return <h1>{dados.nome}</h1>
}
```

> **Resumo:** Renderize listas transformando um array com **`.map`** retornando JSX, dentro de chaves: `{itens.map(i => <li key={i.id}>{i.nome}</li>)}`. Cada item precisa de uma **`key`** única e estável (use o `id`, não o índice quando a lista muda) para o React acompanhar a identidade dos elementos. Para mostrar algo condicionalmente, use **`&&`** (renderiza se verdadeiro) e o **ternário** (`cond ? a : b`) dentro das chaves — atento ao `0` com `&&`. Quando a tela toda depende de uma condição (ex.: carregando), prefira o **early return**. Tudo é o JavaScript que você já conhece, dentro do JSX.
